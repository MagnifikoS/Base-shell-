# Agent: QA

## Mission
Guarantee backward compatibility + BL + Relevé correctness with deterministic tests. Nothing ships without passing this agent's test suite.

## Test Categories

### 1. Edge Function Tests
- [ ] `document_mode` absent → defaults to Facture behavior (backward compatible)
- [ ] `document_mode="facture"` → returns `ExtractionResponse` (unchanged shape)
- [ ] `document_mode="bl"` → returns `BLExtractionResponse` with `needs_human_review: true`
- [ ] `document_mode="releve"` → returns `ReleveExtractionResponse` with `needs_human_review: true`
- [ ] `document_mode="auto"` → classifies then routes to correct extraction
- [ ] Auto mode with clear Facture keywords → routes to Facture
- [ ] Auto mode with clear BL keywords → routes to BL
- [ ] Auto mode with clear Relevé keywords → routes to Relevé
- [ ] Auto mode: document with multiple "Facture N°" references → routes to Relevé (not Facture)
- [ ] Invalid `document_mode` value → returns error

### 2. Classification Tests
- [ ] "FACTURE N° 12345" → `{ doc_type: "facture", confidence: >0.8 }`
- [ ] "BON DE LIVRAISON" → `{ doc_type: "bl", confidence: >0.8 }`
- [ ] "RELEVÉ DE FACTURES" → `{ doc_type: "releve", confidence: >0.8 }`
- [ ] "Récapitulatif des factures du 01/01 au 31/01" → `{ doc_type: "releve", confidence: >0.8 }`
- [ ] Document with 5+ "Facture N°" lines → `{ doc_type: "releve", confidence: >0.7 }`
- [ ] "Solde précédent: 1500.00€" → boost relevé score
- [ ] Empty text → `{ doc_type: "unknown", confidence: <0.3 }`
- [ ] Mixed signals → `{ doc_type: "unknown", confidence: <0.5 }`

### 3. Sanitization Tests
- [ ] Facture sanitize: existing behavior unchanged (run ALL existing tests)
- [ ] BL sanitize: preserves lines without price
- [ ] BL sanitize: preserves lines with null quantity (marks as unreadable)
- [ ] BL sanitize: rejects malformed numeric formats
- [ ] BL sanitize: confidence scores clamped to [0, 1]
- [ ] BL sanitize: unreadable_fields always returns array
- [ ] Relevé sanitize: preserves lines with missing amounts
- [ ] Relevé sanitize: validates date range format
- [ ] Relevé sanitize: handles negative amounts (credit notes / avoirs)
- [ ] Relevé sanitize: normalizes DD/MM/YYYY to YYYY-MM-DD

### 4. Guardrails Tests
- [ ] Facture guardrails: existing behavior unchanged
- [ ] BL guardrails: `missing_quantity` flag when qty is null

### 5. Reconciliation Tests (Relevé-specific)
- [ ] Exact match: same invoice_number + same amount → status "exact_match"
- [ ] Amount mismatch: same invoice_number, different amount → alert
- [ ] Invoice in relevé but not in DB → critical alert "invoice_not_in_db"
- [ ] Invoice in DB but not in relevé → warning "invoice_not_in_releve"
- [ ] Credit note (avoir) handling: positive amount + is_credit flag
- [ ] Balance discrepancy: total_releve ≠ total_db → alert
- [ ] Fuzzy matching: "FA001" matches "FA-001" and "FA 001"
- [ ] Supplier name fuzzy match: "METRO" matches "Metro Cash & Carry"
- [ ] Reconciliation is read-only: no DB writes to `invoices` table
- [ ] Period inference from line dates when header period is null
